# 中文个人笔记网站设计

日期：2026-05-31  
域名：weixianmanbu.shop  
服务器：115.29.175.216，2 核 2G  

## 目标

建设一个中文个人笔记网站，把指定飞书 Wiki 空间或父页面下的笔记定期同步到网站，并允许管理员在网站后台新建本地 Markdown 笔记。

网站面向两类使用者：

- 访客：阅读公开笔记、搜索内容、浏览标签和最近更新。
- 管理员：登录后台，新建和编辑网站本地笔记，查看飞书同步状态，管理公开内容。

第一版不做多人协作、不做双向同步、不把网站新建笔记回写飞书。

## 范围

第一版包含：

- 中文前台网站：首页、笔记列表、笔记详情、标签页、搜索、关于页。
- 单管理员后台：账号密码登录、总览、全部笔记、本地 Markdown 编辑器、飞书同步状态、站点基础设置。
- 飞书同步：从一个指定 Wiki 空间或父页面定期拉取公开内容。
- 本地存储：同步后的正文、元数据、图片和附件落到服务器本地。
- Docker Compose 部署：应用、反向代理、持久化数据目录和备份目录。
- HTTPS：通过反向代理为 `weixianmanbu.shop` 提供 HTTPS。

第一版不包含：

- 多用户权限。
- 评论、点赞、订阅、访问统计等社区功能。
- 飞书和网站的双向同步。
- 复杂主题市场或可视化页面搭建器。
- 移动 App。

## 推荐架构

采用一个轻量全栈应用：

- Next.js 负责前台页面、后台页面、API 路由和服务端渲染。
- SQLite 存储笔记、标签、同步记录、管理员会话和站点设置。
- 本地文件目录存储飞书同步下来的图片和附件，以及后台上传的资源。
- 定时任务负责飞书同步。
- Docker Compose 负责部署。
- Caddy 作为反向代理，自动处理 HTTPS 和域名访问。

访客访问网站时只读取本地数据库和本地资源文件，不实时访问飞书接口。飞书接口只参与同步任务，所以飞书接口异常不会影响公开网站访问。

## 内容模型

核心实体是笔记。

笔记字段：

- `id`：内部 ID。
- `source_type`：`feishu` 或 `local`。
- `source_id`：飞书页面 ID，本地笔记为空。
- `title`：中文标题。
- `slug`：访问路径标识。
- `summary`：摘要，可自动截取或后台编辑。
- `content_markdown`：本地 Markdown 或飞书转换后的 Markdown。
- `content_html`：渲染后的 HTML 缓存。
- `status`：`public`、`draft`、`archived`、`removed`。
- `tags`：标签集合。
- `parent_id`：用于保留飞书 Wiki 层级或站内层级。
- `source_updated_at`：飞书原文更新时间。
- `synced_at`：网站同步时间。
- `created_at` 和 `updated_at`：站内时间。

标签字段：

- `id`
- `name`
- `slug`

同步记录字段：

- `id`
- `status`：成功、失败、部分成功、运行中。
- `started_at`
- `finished_at`
- `message`
- `stats`：新增、更新、下架、失败数量。

访问路径规则：

- 第一版使用稳定短路径，例如 `/notes/<id>`。
- 中文标题保留在页面标题、搜索结果和分享卡片中。
- 不强制把中文标题转拼音 slug，避免标题变更导致链接失效。

## 飞书同步规则

管理员在站点设置里配置一个飞书 Wiki 空间或父页面作为公开同步源。该范围下的内容默认公开。

同步任务执行时：

1. 读取配置的飞书 Wiki 范围。
2. 获取页面树，保留层级关系。
3. 拉取每个页面的标题、正文、更新时间、图片和附件。
4. 将飞书内容转换为网站内部 Markdown 和 HTML。
5. 把图片和附件下载到服务器本地资源目录。
6. 按飞书页面 ID 对应已有笔记，新增或更新数据库记录。
7. 标记本次同步范围里已经不存在的飞书笔记为 `removed`，前台不再展示。
8. 写入同步日志和统计结果。

飞书页面更新后，下一次同步会覆盖网站里对应的飞书来源内容。网站本地 Markdown 笔记不参与飞书同步，也不会被覆盖。

如果某篇飞书页面被删除或移出公开范围，网站不直接硬删除，而是标记为下架或失效，便于后台确认和恢复。

## 本地笔记规则

管理员可以在后台新建 Markdown 笔记。

本地笔记支持：

- 标题。
- 标签。
- 摘要。
- Markdown 正文。
- 实时预览。
- 草稿保存。
- 发布为公开笔记。
- 下架或归档。

本地笔记只存在网站数据库和资源目录，不回写飞书。

## 前台体验

网站是中文个人知识库风格，不做营销型首页。

前台页面：

- 首页：站名、简介、搜索框、最近更新、标签入口、最新公开笔记。
- 笔记列表：按更新时间展示公开笔记。
- 标签页：展示标签列表和每个标签下的笔记。
- 搜索页：支持中文标题、摘要和正文搜索。
- 笔记详情页：中文长文阅读优先，稳定展示标题、更新时间、来源、标签、正文、图片、代码块、引用和列表。
- 关于页：介绍网站和作者，第一版作为一篇特殊的本地 Markdown 页面维护。

前台所有文案默认中文，包括空状态、错误提示、按钮、日期和搜索提示。

第一版站点显示名默认使用 `未闲漫步`，可在后台站点设置里修改。

中文搜索第一版使用 SQLite 的简单关键词匹配，范围包括标题、摘要和正文。搜索目标是能稳定找到中文关键词，不做复杂分词、相关性排序或拼音搜索。

## 后台体验

后台只服务一个管理员。

后台页面：

- 登录页：管理员账号密码登录。
- 总览：公开笔记数、本地草稿数、上次同步时间、最近同步状态。
- 全部笔记：按来源、状态、标签、更新时间筛选。
- 新建笔记：Markdown 编辑器和预览。
- 编辑本地笔记：修改标题、标签、摘要、正文和状态。
- 飞书同步：查看同步配置、上次同步结果、失败原因、同步日志，手动触发同步。
- 标签管理：查看和合并标签。
- 站点设置：站名、简介、关于页、飞书同步源配置。

后台所有界面使用中文。

## 安全设计

后台使用单管理员账号密码登录。

安全要求：

- 管理员密码只保存哈希，不保存明文。
- 会话密钥、飞书应用凭证、数据库路径等敏感配置放在环境变量中。
- `.env` 文件不提交到代码仓库。
- 后台路由需要登录后访问。
- 同步日志和错误详情只在后台展示。
- 前台 API 只返回公开笔记。
- 反向代理启用 HTTPS。

第一版不做复杂权限矩阵。

## 错误处理

飞书同步失败时：

- 不影响前台访问。
- 保留上一次成功同步后的公开内容。
- 后台显示失败时间、失败阶段和错误摘要。
- 支持管理员手动重新同步。

内容转换失败时：

- 单篇失败不阻断整次同步。
- 失败页面记录到同步日志。
- 其他成功页面仍然更新。

图片或附件下载失败时：

- 正文仍保存。
- 对应资源位置显示中文错误提示，不在前台暴露飞书私有资源链接。
- 后台同步日志提示资源下载失败。

数据库或文件写入失败时：

- 同步任务停止并记录错误。
- 前台继续使用已有内容。

## 部署设计

使用 Docker Compose 部署到 `115.29.175.216`。

服务组成：

- `app`：Next.js 应用。
- `proxy`：Caddy，负责 `weixianmanbu.shop`、HTTPS 和静态资源转发。

持久化目录：

- `data/`：SQLite 数据库。
- `uploads/`：图片、附件和后台上传资源。
- `backups/`：数据库和资源备份。

部署后需要：

- 域名 `weixianmanbu.shop` 的 A 记录指向 `115.29.175.216`。
- 服务器开放 80 和 443 端口。
- 配置飞书应用凭证。
- 配置管理员初始账号和密码。
- 配置定时同步频率。

## 备份策略

第一版采用简单文件备份：

- 定期备份 SQLite 数据库。
- 定期备份 `uploads/` 资源目录。
- 备份文件写入 `backups/`。
- 后续可扩展为自动上传到对象存储或另一台服务器。

备份不包含 `.env` 明文文件。敏感配置需要单独安全保存。
